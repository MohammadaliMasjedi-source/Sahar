# Sample contribution templates

*Four ready-to-copy templates, one per common contribution shape. Every file here is **structure
only** — placeholder text, checkboxes, and instructions. None contains lesson content, and none
should ever be filled in with anything a child would read/hear/see without going through
[`../REVIEWER-GATE.md`](../REVIEWER-GATE.md) first, exactly like every other content path in this
scaffold.*

| Template | Use it for |
|---|---|
| [`PR-DESCRIPTION-TEMPLATE.md`](PR-DESCRIPTION-TEMPLATE.md) | Any pull request — copy this into the PR description box. |
| [`MODULE-FILL-REQUEST-TEMPLATE.md`](MODULE-FILL-REQUEST-TEMPLATE.md) | Proposing to fill in part of an age-band module skeleton (`../modules/`) — explicitly marked `needs-reviewers`. |
| [`TRANSLATION-TASK-TEMPLATE.md`](TRANSLATION-TASK-TEMPLATE.md) | Proposing or logging a translation task (fixing existing `fa`/`en`/`de`, or adding a new language). |
| [`ACCESSIBILITY-CHECK-TEMPLATE.md`](ACCESSIBILITY-CHECK-TEMPLATE.md) | Recording an accessibility audit pass — screen reader, contrast, keyboard nav, RTL. |

## How to use one

1. Copy the template's raw content.
2. Fill in the bracketed placeholders — `[like this]` — with your real information.
3. Leave every unchecked box unchecked until it's genuinely true; don't pre-check boxes to look
   finished. A reviewer trusts an honestly partial checklist far more than a suspiciously complete
   one, and this project's whole culture (`SAHAR-COVERAGE-STATUS.md`) is built on that honesty.
4. Attach it to your PR, issue, or the relevant tracking file.

## Why these four

They map to the highest-traffic, highest-value entry points named across this scaffold:
- **Every** contribution needs a PR description — one universal template covers it.
- **Content-pack authoring** (`DOD-BY-TASK-TYPE.md` §3) is the task type with the most structure
  around it, and the one most likely to be started by someone unfamiliar with the gate — a
  ready-made `needs-reviewers` request removes any excuse to skip labeling it.
- **Translation** (`DOD-BY-TASK-TYPE.md` §2) is named in `CONTRIBUTING.md` as one of the two
  highest-leverage ways to help (alongside being a reviewer) — a template lowers the bar to start.
- **Accessibility** (`DOD-BY-TASK-TYPE.md` §7) is new to this scaffold and benefits most from a
  template, since "what did you actually check" is otherwise easy to state vaguely.
